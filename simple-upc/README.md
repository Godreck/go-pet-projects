**UPC** - Universal Product Code, набор чёрных полос, которые представляют последовательность из 95 бит.

Биты сгруппированы по оределённому принципу:
- 3 бита: Левый шаблон-ограничитель (всегда 101)
- 6х7 бит: Цифры слева
- 5 бит: Центральный шаблон-огранничитель (всегда 01010)
- 6х7 бит: Цифры справа
- 3 бита: Правый шаблон-ограничитель (всегда 101)

Всего в коде UPC зашифровано 12 десятичный цифр, для расшифровки используется таблица:

|-|-|
|0011001|1|
|-|-|
|0010011|2|
|-|-|
|0111101|3|
|-|-|
|0100011|4|

Все левосторонние коды начинаются в 0 и заканчиваются 1.

*Контроль чётности*: В каждом из кодов нечётное количество единиц, благодаря чему можно проверить код на корректность.
*Чётный паритет* - чётное количество битов-единиц в коде.
*Нечётный паритет* - нечётныое количество битов-единиц в коде.

**Правосторонние коды**
Дополняют коды из предыдущей таблицы, получить правосторонние коды можно инвертировав соответствующий *левосторонний код*, таким образом там где в левосторонних кодах был 0, будет стоять 1, и наоборот.

**Значение цифр кода**
- Первая цифра характеризует тип кода (например, 0 - обычный, 5 - скидка)
- 2-6 цифры определяют код Производителя.
- 7-11 цифры опредеяют код конкретного продукта или товара (например макароны или авторучка)
- 12 цифра - это символ проверки остатка, которая используется для исключения ошибок.

**Проверка на остаток**
Для этого нужно присвоить букву каждой из первых 11 цифр (для примера возьмём 0 51000 01251), последняя цифра у этого кода - 7.

Получаем: A BCDEF GHIJK
Вычислим: 3 * (A + C + E + G + I + K) + (B + D + F + H + J)
Результат вычтем из ближайшего большего числа, кратного десяти, полученное число и должно быть символом остатка: 3 * (0 + 1 + 0 + 0 + 2 + 1) + (5 + 0 + 0 + 1 + 5) = 3 * 4 + 11 = 23.
Ближайшее число кратное десяти это 30, тогда 30 - 23 = 7. Всё верно, проверка пройдена.

**Удобство чтения UPC**
Штрихкод можно считывать в обоих направлениях. Если в первых считанных цифрах количество единиц чётно, сканер распознаёт, что код читается справа налево. Для расшифровки правосторонних цифр компьютер использует таблицу:
*Правосторонние коды в обратном направлении*

|Код|Цифра|
|-|-|
|0100111|0|
|-|-|
|0110011|1|
|-|-|
|0011011|2|
|-|-|
|0100001|3|
|-|-|
|0011101|4|
|-|-|
|0111001|5|
|-|-|
|0000101|6|
|-|-|
|0010001|7|
|-|-|
|0001001|8|
|-|-|
|0010111|9|

Левосторонние коды в обратном направлении получаются инвертированием правосторонних кодов обратного направления.

Эти 7-битовые коды отличаются от кодов, считываемых слева направо и, как следствие, никакой путаницы не возникает.
